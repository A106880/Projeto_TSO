Notas:
-Os ficheiros devem sempre ter tamnahos multiplos de 4kb.
-Nao é permitido alterar conteudo dos ficheiros ja existentes(dentro do tamanho original), apenas é permitido adicionar conteudo mas sempre em blocos de 4kb
        -Exemplo: Ficheiro tem "abcd", nao podemos mudar o ficheiro para "adbc" apenas posso fazer "abcdefgh"


Estruturas de dados
-criar uma struct A que teria um array de id das subpastas e tambem um array de ids de ficheiros
-struct B que tem o id do ficheiro, o tamanho original associado e os blocos que utiliza de forma ordenada(usar list)
-hash table em que a key é a pasta e o value seria a struct A
-hashtable com a key o id do ficheiro e como value a struct B
-hash table em que cada key é a identificacao do bloco e o value é um contador de ficheiros que necessitam do bloco
    - ao remover ficheiros é diminuido o contador nos blocos que o ficheiro em questao utliza.
    - se o array de ficheiros do bloco ficar vazio, a entrada desse bloco deve ser apagada
-hashtable de partial index, ou seja, ter uma hashtable que sirva para manter em memoria os blocos mais utilizados
